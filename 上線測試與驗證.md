# 新港國小英語單字王 — 上線測試與驗證清單

本文件用於把目前專案部署到 Supabase + GitHub Pages 後，完成第一輪 MVP 測試。

## 1. Supabase 資料庫

在 Supabase SQL Editor 執行：

```sql
-- 你已經有 english-word-king / tzvnyluqommusppbzyiy 專案：
-- 優先執行 supabase/production-upgrade-existing-project.sql

-- 只有全新空專案才直接執行：
-- supabase/migrations/20260627000000_init.sql

-- 若是已建立過舊版資料庫，至少確認以下物件存在：
-- student_roster, students, user_roles, challenge_sessions, daily_attempts,
-- system_config, activity_feed, school_leaderboard, student_leaderboard
```

重點檢查：
- `user_roles` 必須存在。
- `system_config` 不應開放 anon/authenticated 直接讀取。
- `challenge_sessions` 允許 teacher/admin 建立與關閉自己的場次。
- `daily_attempts.practice = true` 的課堂挑戰不會計入首頁學校總分。

## 2. Edge Functions

使用 Supabase CLI 部署兩支函式：

```bash
supabase functions deploy generate-challenge
supabase functions deploy admin-api
```

`supabase/config.toml` 已將這兩支 function 設為：

```toml
verify_jwt = false
```

這是為了讓 GitHub Pages 網站的瀏覽器 CORS preflight (`OPTIONS`) 可以進入函式並取得 CORS 回應。安全性仍由函式內部自行驗證 `Authorization: Bearer <Supabase user JWT>`；未登入或非授權帳號仍會被拒絕。

確認 Project Secrets 已存在：

```bash
SUPABASE_URL
SUPABASE_SERVICE_ROLE_KEY
```

AI 金鑰可部署後從 `admin.html` 設定；未設定時 `generate-challenge` 會自動使用內建題庫 fallback。

## 3. Auth 設定

Supabase Dashboard → Authentication → URL Configuration：
- Site URL 設為 GitHub Pages 網站網址。
- Redirect URLs 加入 GitHub Pages 的 `join.html` 完整網址。
- 本機測試可加入 `http://127.0.0.1:3000/join.html` 與 `http://localhost:3000/join.html`。

GitHub Pages 專案頁通常會長這樣：

```text
https://你的帳號.github.io/english-word-king/
```

所以 Redirect URL 要填完整的：

```text
https://你的帳號.github.io/english-word-king/join.html
```

Google OAuth：
- 啟用 Google provider。
- 確認學校帳號網域為 `@gapp.hcc.edu.tw`。

## 3.5 GitHub Pages

本專案使用 GitHub Actions 發布 GitHub Pages：

1. 將專案推到 GitHub repository 的 `main` branch。
2. 到 GitHub repository → Settings → Pages。
3. Source 選 `GitHub Actions`。
4. `.github/workflows/pages.yml` 會執行：
   - `npm run check`
   - `npm run build:pages`
   - 發布 `dist/` 到 GitHub Pages

`dist/` 只包含網站需要的 HTML/CSS/JS/data 與小型 PDF，不會發布 `supabase/` 後端原始碼、工具腳本或大型教材 PDF。

## 5. 學生端驗收

1. 名冊內學生登入後應進入 `app.html`。
2. 名冊外學生登入應顯示找不到帳號。
3. 每日挑戰可取得 12 題並完成。
4. 沒有 AI key 時仍可取得內建 fallback 題目。
5. 每日挑戰完成後：
   - `daily_attempts.practice = false`
   - `students.total_score` 增加
   - `students.mastery` 更新
   - `activity_feed` 由資料庫 trigger 自動新增一筆跑馬燈動態
   - 首頁排行榜更新

## 6. 教師端與課堂挑戰驗收

1. 教師登入後進入 `teacher.html`。
2. 建立 6 位數場次碼。
3. 學生在 `app.html` 切到課堂挑戰並輸入場次碼。
4. 學生完成後教師端即時榜出現資料。
5. 課堂挑戰紀錄應為：
   - `daily_attempts.session_id` 有值
   - `daily_attempts.practice = true`
   - 不消耗學生每日計分挑戰額度
